# BoxPlayer v5.0.4

> 发布主题：修复 Linux 自定义 MPV 播放、完善 115 网盘认证播放链路。

## 发布摘要

v5.0.4 是基于 v5.0.3 的播放修复版本，重点处理用户反馈的 Linux 环境视频无法播放，以及 115 网盘播放链接需要认证 header 时无法稳定播放的问题。

本版本统一收口了“获取播放链接时返回的 headers”和“实际播放器请求时使用的 headers”之间的链路，降低 115 网盘转码视频、原画、字幕在内置网页播放器和自定义 MPV 中播放失败的概率。

## 重点修复

- 修复 Linux 上自定义 MPV 播放器无法打开视频的问题。此前在启用播放历史或播放列表时，MPV 会走直接 `spawn` 控制模式，但播放器路径和 URL 参数仍按 shell 方式加引号，导致 `/usr/bin/mpv`、`mpv` 或 URL 被错误传递。
- 修复 Linux 自定义 MPV 填写 `mpv --profile=...`、带空格路径等命令时的解析问题。
- 修复自定义 MPV 只传递 `Authorization` 的问题，现在会将播放链接返回的所有有效 headers 传递给 MPV，并单独处理 `User-Agent`。
- 修复 115 网盘转码播放、原画播放在内置网页播放器中没有使用认证 headers 的问题。
- 修复 115 网盘字幕播放 headers 来源不准确的问题，字幕链接会使用字幕接口请求时对应的 `Authorization` 和 `User-Agent`。
- 修复媒体服务器播放 URL 带 headers 时，网页播放器未通过本地代理附带 headers 的问题。
- 将本地 `OpenList/` 项目目录加入 `.gitignore`，避免误提交本地子项目。

## 支持情况

- 115 网盘：转码视频、原画、字幕会保留播放接口返回的认证 headers，并通过本地代理或 MPV 参数传递给播放器。
- 百度网盘、夸克网盘：继续沿用已有代理补齐 header/cookie 的逻辑。
- Dropbox、OneDrive、Box、光鸭、123、139、189：通常使用可直接访问的临时直链；如后续返回 headers，新的播放链路也会通过本地代理处理。

## 升级建议

- Linux 用户使用自定义 MPV 播放器时建议升级。
- 使用 115 网盘播放转码视频、原画或字幕的用户建议升级。
- 使用需要认证 header 的私有媒体服务器播放源的用户建议升级。
