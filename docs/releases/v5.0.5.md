# BoxPlayer v5.0.5

> 发布主题：AI 工作台与媒体获取正式落地，完善多网盘能力、播放下载体验与生产构建校验。

## 发布摘要

v5.0.5 将资源检索、媒体获取、追更和任务审计收拢到 AI 工作台。用户可以从搜索结果选择目标网盘和目录，由 Agent 在明确的能力边界内检索候选、导入资源、核验落盘并整理媒体库；涉及导入、移动、删除等写入操作均会先生成可确认的计划。

本版本还扩展了多网盘的能力声明与交互边界，增强夸克、115、123、PikPak、光鸭等网盘的目录、离线、分享和播放链路，并针对桌面端目录选择和请求认证补齐了稳定性处理。

## 重点新增

- **AI 工作台**：新增统一的对话、计划确认、工具活动、执行上下文与异步通知界面，支持检索、阅读与受控文件操作。
- **Agent 媒体获取（Pro）**：媒体搜索结果可直接创建获取任务，选择目标网盘及保存目录后，按标题、年份、季集、画质、字幕和目标网盘能力筛选候选并执行导入。
- **任务核验与追更**：获取任务会记录候选、导入、落盘核验、整理和失败原因；电视剧、动画支持缺集巡检、季订阅和追更任务，历史媒体自动补全默认关闭。
- **媒体结果体验**：搜索页新增媒体结果、热门内容和获取入口；媒体详情、媒体库、海报行与全局搜索同步适配新的获取和状态展示。
- **文档能力**：新增文档解析服务和文档 AI 入口，阅读器与书库补充对应的元数据、回收站及检索行为。

## 多网盘与传输改进

- **夸克网盘**：优化 Cookie 归一化和登录态校验；目录列表在 Electron 主进程通过显式 Cookie 发起请求，避免渲染进程跨域请求丢失认证信息。分享、下载、视频预览和目录错误信息也得到补强。
- **目录选择器**：第三方网盘按单根目录模型浏览；阿里云盘目标目录固定使用资源盘根目录，目录懒加载和选择状态更稳定。
- **网盘能力边界**：为阿里、115、123、百度、夸克、PikPak、139、189、Dropbox、OneDrive、Box、光鸭等网盘建立能力声明，菜单、Agent 和离线任务将据此限制到已支持的操作。
- **分享与离线任务**：完善 115、123、PikPak、光鸭的离线/分享处理，增加 123 付费分享提示，并改善 BT 目标目录、任务状态和 aria2 参数处理。
- **下载与本地播放**：下载记录、任务详情、BT 文件选择及本地视频播放策略继续完善；媒体播放链路补齐 MPV 启停与代理请求处理。

## 构建与运行时

- 发布工作流新增 Linux MPV 校验，在跨平台打包前验证原生播放运行时。
- 生产构建启用原生依赖重建，并将 `better-sqlite3` 作为运行时依赖，提升桌面包内 SQLite 运行可靠性。
- 新增 Agent 运行时依赖与相关测试覆盖，继续使用 SQLite 保存检索、任务和追更状态。

## 升级说明

- AI 媒体获取依赖目标网盘已开放的分享导入、磁力离线或 HTTP/HTTPS 离线能力；资源可用性、账户状态和网盘限制会影响最终结果。
- 已登录的夸克用户如仍收到 `require login [guest]`，请重新登录夸克以刷新 Cookie；桌面端更新后需完整退出并重启应用，确保主进程与 preload 更新生效。
- 追更巡检默认时间为北京时间 `06:00,21:00`，历史媒体自动补全需在设置中显式开启。
